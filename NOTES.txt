Types of CSS in Web Development

1. Inline CSS

Inline CSS is applied directly to a specific HTML element using the
style attribute.Usage: Best for quick, one-off style changes or testing
specific elements.

Where and How to use: It's used directly within the opening tag of element using style attribute

---------------------------------------------------------------------------

2. Internal CSS

Internal CSS is defined within a "style" tag in the head section of an
keep styles separate from content

Where and How to use: It's used inside head element within style element

---------------------------------------------------------------------------

3. External CSS
External CSS is defined in a separate .css file and linked to the HTML
document using the "<link />" tag.Usage: Ideal for styling multiple pages
consistently and maintaining a clean separation between content and
presentation.

Where and How to use: We need to create separate file and link it with html using <link> tag / element


-----------------------------------------------------------------------------

4. CSS Units

(1.) px (Pixels) : A fixed unit of measurement.
1px = one pixel on the screen (roughly a tiny dot).


.box {
  width: 300px;
  height: 100px;
}

Width → always 300px
Height → always 100px


---------------------------------------------

(2.) % (Percentage)
Takes a percentage of its parent element.

example:
<div class="parent">
  <div class="child"></div>
</div>



.parent{
   width:400px;
}

.child{
   width:50%;
}


-----------------------------

(3). em : Relative to the font-size of its parent.

Example:
.parent {
  font-size: 20px;
}

.child {
  font-size: 2em;
}

Calculation: 2 × 20px = 40px


----------------------------------------

(4). rem : Relative to the root (html) font-size
---------------------------------------------------


(5). vh (Viewport Height) : Percentage of browser height.

Example:
.hero {
  height: 100vh;
}


(6). vw (Viewport Width) :Percentage of browser width.

Example:
.banner {
  width: 50vw;
}

BOX SIZING
The CSS box-sizing property defines how to calculate the width and height of an element: should the calculation include padding and borders, or not.


Default Behaviour:

{
  box-sizing:content-box
}

width + padding + border = actual width of an element


height + padding + border = actual height of an element


The `box-sizing` property solves this problem.

{
  box-sizing: border-box
}

What is Git and Why Do We Need Version Control

When people work on software, documents, websites, or any project files,
problems quickly appear:



1. It becomes hard to track who changed what
2. Older versions(work) are lost
3. Collaboration becomes confusing
4. Recovering from mistakes is difficult



#How to tackle those challenges:
This is why Version Control Systems (VCS) were created.

#Version control system:
A Version Control System is a tool that:

Tracks file changes over time
Stores history of modifications
Allows multiple people to collaborate
Helps recover previous versions


------------------------------------------
Difference Between Git and GitHub

#Git:
Git is a free and open-source version control tool created by Linus Torvalds.
It works locally on your computer and helps developers:

Track code changes
Create branches
Merge code
Revert mistakes
Maintain project history


#Github:
GitHub is a cloud-based platform that hosts Git repositories online.

Features:
1. Remote repository storage
2. Team collaboration
3. Push/Pull requests
4. Code review
5. CI/CD integration (Github Actions)


Git as a 'Decentralized Version Control System' (DVCS) means every developer has a complete copy of the project and its history on their own computer.

Unlike a centralized system where one main server stores everything, Git lets each user work independently.

Downloads changes AND merges them into your current branch


9. git push

11. Tagging
Tags in Git are used to mark specific points in a repository's history,
often used for marking --release-- points like v1.0, v2.0, etc

a. git tag tag_name (Eg. v1,v2)
b. git tag tag_name commitHash
c. git tag tag_name -a "message" commitHash
d. git tag :  To lists down all Tags
e. git push origin v1.0 : to push one tag
f. git push origin --tags :  TO push all tags
g. git tag -d v1 : To delete tag
